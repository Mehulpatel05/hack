# 🎬 CampaignX - 3 Minute Demo Script

## Setup Before Recording
- Backend running on localhost:3001
- Frontend running on localhost:3000
- Browser window ready
- Screen recorder ready (OBS/Loom)

---

## ⏱ 0:00 - 0:30 | Brief Input & Parsing

**Action:**
1. Show clean UI with CampaignX header
2. Paste campaign brief:
   ```
   Run email campaign for launching XDeposit fixed deposit product. 
   Target new customers aged 25-50. Offer 5% interest rate. 
   Include link to superbfsi.com/xdeposit. 
   Optimize for open rate and click rate.
   ```
3. Click "Generate Strategy →"

**Narration:**
> "CampaignX is an AI multi-agent system for autonomous email marketing. 
> I'll input a natural language brief about launching XDeposit, 
> a new fixed deposit product from SuperBFSI."

**Show:**
- Brief input textarea
- Loading state
- Parsed brief appears with extracted fields

---

## ⏱ 0:30 - 1:00 | Strategy Generation

**Action:**
1. Review parsed brief showing:
   - Product: XDeposit
   - Target: new_customers
   - Offer: 5% interest rate
   - Objectives: open_rate, click_rate
2. Click "✓ Approve & Generate Strategy"
3. Show generated strategy

**Narration:**
> "The Brief Parser Agent extracted all key information. 
> Now the Strategy Agent generates campaign parameters: 
> target segments, A/B test variants, and optimal send time."

**Show:**
- Parsed brief card
- Strategy card with:
  - Segments: age_25_35, age_36_50
  - Variants: 2
  - Send Time: 10:00 AM
  - A/B Test: Yes

---

## ⏱ 1:00 - 1:30 | Content Generation & Preview

**Action:**
1. Click "✓ Approve & Generate Content"
2. Show 2 email variants:
   - Variant A: Emoji-rich, exciting
   - Variant B: Professional, clean
3. Expand HTML preview for one variant
4. Click "🚀 Approve & Launch Campaign"

**Narration:**
> "The Content Agent creates multiple email variants. 
> Variant A uses emojis for engagement, 
> Variant B is more professional. 
> Both include personalized CTAs and the offer details."

**Show:**
- Email preview cards
- Subject lines
- CTA buttons
- HTML preview

---

## ⏱ 1:30 - 2:00 | Dynamic API Call & Launch

**Action:**
1. Show browser console (optional)
2. Display API call logs showing:
   - OpenAPI spec parsing
   - Dynamic endpoint generation
   - Campaign scheduled response

**Narration:**
> "Here's the key differentiator: the API Agent dynamically parses 
> the OpenAPI specification and generates API calls on-the-fly. 
> No hardcoded endpoints. The campaign is now scheduled."

**Show:**
- Console logs with API parsing
- Campaign scheduled confirmation
- Campaign ID generated

## ⏱ 2:00 - 2:30 | Performance Analytics

**Action:**
1. Show performance dashboard with:
   - Open Rate: 25.5%
   - Click Rate: 3.2%
   - Overall Score: 10.9
2. Highlight weighted scoring formula
3. Show detected issues:
   - Low click rate
   - Poor performance in age_25_35 segment

**Narration:**
> "The Analytics Agent fetches real-time metrics. 
> The scoring uses 70% weight for click rate and 30% for open rate. 
> It detected low click rates in the 25-35 age segment."

**Show:**
- Metrics dashboard
- Score calculation
- Issues detected section

---

## ⏱ 2:30 - 3:00 | Autonomous Optimization Loop ⚡ CRITICAL!

**Action:**
1. Click "🔄 Auto-Optimize Campaign"
2. Show optimization suggestion:
   - Target Segment: age_25_35
   - New Subject: "💰 Limited Time - 5% Interest on XDeposit"
   - New CTA: "Start Earning Today"
   - Send Time: 7:00 PM
   - Reasoning: "Younger audience responds better to urgency and evening sends"
3. Click "✓ Approve & Relaunch"
4. Show success message

**Narration:**
> "This is the autonomous optimization loop in action. 
> The Optimization Agent detected the issue, 
> generated an improved variant targeting the low-performing segment, 
> and suggested a better send time. 
> After human approval, it automatically relaunches the campaign. 
> This cycle continues until optimal performance is achieved."

**Show:**
- Optimization card
- New subject line and CTA
- Reasoning explanation
- Approval and relaunch
- Success confirmation

## 🎥 Recording Tips

1. **Practice 2-3 times** before final recording
2. **Speak clearly** and at moderate pace
3. **Highlight key features**: Dynamic API, Optimization Loop
4. **Show, don't just tell**: Let the UI demonstrate capabilities
5. **Keep energy high**: Enthusiasm matters!
6. **Time each section**: Use timer to stay under 3 minutes
